# قاعدة صارمة للفحص الشامل (Strict Full-Scan Rule)

## الهدف
ضمان عدم تخطي أي ملف أثناء التدقيق، وأن كل نتيجة فحص تكون مبنية على تغطية كاملة قابلة للإثبات.

## القاعدة الإلزامية
لا يُسمح بإصدار حكم تدقيقي (Pass/Fail/Gap) قبل تنفيذ قراءة كاملة لكل الملفات من أول بايت إلى آخر بايت.

## آلية التنفيذ الإلزامية
1. تنفيذ Full Byte-Stream Read على كامل الـWorkspace.
2. إنتاج تقرير تغطية يحتوي:
   - عدد الملفات المكتشفة.
   - عدد الملفات المقروءة بنجاح.
   - عدد ملفات الفشل (إن وجدت).
   - إجمالي حجم البيانات المقروءة.
   - Manifest لكل ملف مع SHA256.
3. إنتاج ملخص حسب كل فولدر (Files/Bytes).
4. أي فشل قراءة = إيقاف التدقيق حتى معالجة السبب أو توثيق استثناء صريح.

## سياسة الاستمرارية
عند كل جلسة تدقيق جديدة:
- يُعاد توليد التقريرين بنفس اليوم/التاريخ.
- لا يُعتمد أي استنتاج جديد بدون نسخة Coverage حديثة.

## ملاحظة منهجية
القراءة البايتية تثبت عدم تخطي الملفات، ثم يتم التدقيق الدلالي/المحاسبي على مراحل حسب الأولوية والمخاطر.
